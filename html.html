What is HTML ?
  • HyperText Markup Language is a markup language for displaying
  • HTML structures the content (words, images, video, etc.) of your web page.
  • HTML is interpreted and displayed by a web browser.
  • HTML uses elements, consisting of tags, to describe the semantics, or meaning, of the content.
  • Different browsers may render HTML differently.
Elements
  • HTML Element are the building bocks of HTML documents.
    • Element and Tags are sometimes used synonymously in the industry
  • An HTML Element is made of
    • A Start Tag/Opening Tag: Tag Name enclosed between "<" and ">"
    • Content: Text (or some other element)
    • End Tag/Closing Tag: Text preceded by "/" and enclosed between "<" and ">"
<!DOCTYPE html public "-//w3c//dtd html 4.01//en"
   "http://www.w3.org/TR/html4/strict.dtd">
<html>
   <head>
      <title>My first HTML document</title>
   </head>
   <body>
      <p>Hello world! </p>
   </body>
</html>
Element Attributes
  • Attributes provide additional information about the HTML elements
  • They are part of the start tag and comprise of name-value pairs
  • Attribute names are always case-insensitive.
  • Attribute values are generally case-insensitive and are always enclosed in quotes
    • If the value contains double quotes then it is imperative to use single quote for the value

Empty vs. Container Tags
  • Refresher: Element has Start Tag, Content, End Tag
    • Element and Tags are sometimes used synonymously in the industry
  • Container tag: acts as a container for content or for other tags
    • Closing tag is mandatory for container tag
  • Empty tag: has no content/tags to operate on
    • Consequently, closing tag is not mandatory for empty tags

Case Sensitivity in HTML
  • HTML is case insensitive with some exceptions
    • The values of IDs, classes, element names, attribute names, and attribute values may be handled as case-sensitive.
    • According to the practice, we follow lower case for HTML tags.
Semantics
  • SEMANTICS = MEANINGS.
  • Elements, attributes, and attribute values in HTML all have specific semantics.
  • HTML documents can be presented in many different ways
    • With a web browser on a desktop PC.
    • With a tablet device, such as an iPad
    • For a screen reader for someone who is visually impaired
  • HTML can present content on many different types of devices because the markup conveys they meaning of the content, not how it should be displayed
Page Structure
  • An HTML webpage is made up of three main parts:
    • Document Type Declaration (<!DOCTYPE html>)
      • A single tag which contains information about the markup language used in the document
      • The HTML5 Doctype Declaration is simply <!DOCTYPE html>
    • Header (<head>)
      • Contains information about the web page
    • Body (<body>)
      • Contains the content for the web page

List Elements
  • Lists group lists of information, and contain at least one list element.
  • There are three types of lists in HTML:
    • Unordered List (<ul>)
      • An unordered list contains list elements where the order is not important. They are generally displayed with some type of bullet character.
    • Ordered List (<ol>)
      • An ordered list contains list elements where the order is important. They are generally numbered in some manner when displayed.
    • Definition List (<dl>)
      • A definition list generally contains term/definition pairs.
  • Lists can be nested, even between different types of lists.
  • Lists are also frequently used for navigation menus.

    How HTML Forms Work
    • When a form is submitted, input fields in the form are passed to the server as key value pair.
    • The user submits a form. The data is sent to the server
    • Data is processed at the server
    • Server returns the result of data processing and result is displayed as Html page.
    Action and Method Attributes
    • Form has two important attributes which facilitates communication with server.
      • Action
      • Method
    • Action attribute specifies the url of the server which processes the form data.
    • Data is processed at the server
    • Method attribute specifies how the data will be passed to the server
      • get
      • post

    Get vs. Post
    • Get
      • Data is appended to the url as query string
      • Mostly used by search engine
      • can be bookmarked
      • There is upper limit on the length of query string
    • Post
      • Data is not appended to the url. It's sent as part of the request body
      • There is no limit on the data to be sent
    • As a general rule, post method should be used unless there is a need to bookmark the page
